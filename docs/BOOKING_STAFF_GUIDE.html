📧 Guest Booking + Token Access Guest bookings now generate a secure access token so guests can view/manage their appointment via a unique link — without registering.
🏗️ Force-Update Guard All clients auto-refresh when a new app version is deployed. Caches, service workers, and stale localStorage are cleared automatically, preserving auth sessions.

1. System Overview

The TestD booking system allows clients to schedule appointments at any of the four SWING branches. It supports registered users, guest users (email only), and walk-in clients.

Key Features

  • Multi-service appointments — Clients select multiple services per visit
  • Density-aware scheduling — Slots show occupancy (🟢 Low / 🟡 Medium / 🔴 High)
  • Atomic booking — Database-level locks prevent double-booking V2
  • Same-day booking — Book for today; only future time slots shown V2
  • Wait-time estimates — Factors counselor count, service weight, walk-in pressure
  • Referral codes — Unique code per booking (e.g. SWG-A3K9X2)
  • Real-time updates — Dashboard auto-refreshes on changes
  • Guest booking — Email only; generates secure access token V2
  • 6-language UI — TH, EN, KH, LA, VI, MY V2
  • Blackout management — Block full days or specific time slots V2
  • Risk assessment — Optional guided questionnaire recommends services

2. User Booking Flow (4-Step Wizard)

Clients access the booking wizard at /booking. A progress bar shows 4 steps.

1 Choose Branch

Card-based selection showing: hero photo, Google Maps rating & review count, address, open days/hours. Direct link to Google Maps.

2 Select Services

Multi-select with checkboxes. Each service shows: icon, name, description, and pricing eligibility badges (🇹🇭 Free Thai NHSO, 🌍 Free Global Fund, 💰 Paid). Optional Risk Assessment quiz auto-recommends services.

3 Pick Date & Time

Horizontal scrollable date chips (30-day range). Only open days shown. Blacked-out dates appear disabled with strikethrough. Today has a pulsing dot. V2

Time slots use the DensityTimeSelector component showing 2-hour blocks with color-coded density and estimated wait times.

4 Confirm & Get Code

Summary of branch, services, date/time, and optional notes. Guest users enter email here. On submit → referral code displayed with copy button and screenshot prompt.

💡 Smart Suggestion If a client picks a high-density time block, the system automatically suggests quieter alternatives.
🩺 Risk Assessment (Optional) V2 On the services step, clients can tap "ไม่แน่ใจ? ทำแบบประเมินความเสี่ยง" to take a guided questionnaire. The system then auto-selects recommended services based on their answers. Questions are configurable in the risk_assessment_questions table.

3. Guest (Anonymous) Booking + Token Access

Clients do not need an account. Only an email address is required.

How It Works:

  1. Client completes the 4-step wizard without logging in
  2. On the confirm step, they enter a contact email
  3. System calls create_anonymous_appointment RPC (server-side, bypasses RLS)
  4. Client receives a referral code on screen
  5. System generates a secure guest access token via generate_guest_access_token RPC V2
  6. A confirmation email is sent with the code + a manage-booking link
  7. Booking is also saved to localStorage for offline reference
✅ Guest Access Token V2 Guests receive a unique link at /guest-appointments?token=... to view and manage their booking — no account needed. Token expires after a configurable period.
✅ Account Linking After booking, guests see: "Want to manage your appointments more easily? Register to view, reschedule, or cancel instantly." If they register with the same email, the system links the booking.

4. Referral Codes

📌 Format: SWG-XXXXXX
Auto-generated by database trigger. Unique per appointment. Both registered and guest bookings receive a code.

For Clients:

  • Displayed prominently with a copy button on the success screen
  • Instruction: "Please take a screenshot and show this to Medical Registration"
  • Code is also included in the confirmation email

For Staff:

  • Go to Admin → Booking tab
  • Use the referral code search box — this bypasses date filters
  • Verify client identity → update status to Arrived
⚠️ Important Always verify the referral code matches the person. Ask the client to show their screenshot or email confirmation.

6. Walk-in Management

Built directly into the Bento Dashboard. No separate page needed.

1 Create Walk-in

Select branch from dropdown → click "+ Walk-in". System creates appointment with source: 'walkin', status: waiting, and assigns a referral code.

2 Start Service

When counselor is ready, click "Start" (▶️). Status changes to in_progress. Arrival time is recorded.

3 Complete Service

After consultation, click "Done" (✅). Status changes to completed. Completion time recorded.

4 Impact on Wait Times

Active walk-ins increase walk-in pressure, causing wait-time estimates to go up for clients booking online.

The dashboard shows an active walk-ins counter badge with live status (Waiting / In Service) for each walk-in.

💡 Quick Register The Admin panel also has a "Quick Register" tab where staff can register walk-in clients via Thai ID OCR scan — no client device or internet needed.

10. Blackout / Holiday Management NEW

Admins can create blackout periods to block bookings for holidays, maintenance, or special events.

Blackout Types:

TypeScopeEffect
All-dayFull day closureDate appears disabled in client date picker with strikethrough. No slots available.
PartialSpecific time rangeOnly affected time slots are blocked. Other slots remain available.
GlobalAll branchesApplies to every branch
Branch-specificSelected branches onlyOnly listed branches are affected

Blackouts are stored in the booking_blackouts table. The get_available_slots RPC checks blackouts and returns day_is_closed, blackout_title, and closure_reason for the UI to display.

💡 Client Experience When a date is blacked out, clients see the date as disabled with a tooltip showing the reason (e.g., "Public Holiday — Songkran"). For partial blackouts, affected time slots show a lock icon.

12. Quick Reference Card

🏥 Daily Workflow for Registration Staff
  1. Open Admin → Booking at the start of each day
  2. Check the Bento Dashboard — review total appointments, busy level, and branch loads
  3. When client arrives → ask for referral code or screenshot
  4. Search code in dashboard → verify details → mark as Confirmed or In Progress
  5. If walk-in → select branch → click "+ Walk-in" → Start when ready
  6. After consultation → mark as Completed → add staff notes
  7. End of day → review no-shows (auto-marked by system cron) and update as needed
🔍 Finding a Booking Quickly
  • By referral code: Search box — bypasses date filter
  • By date: Calendar navigation
  • By branch: Click branch card → timeline view
  • By time: Click peak hour block → filtered view
