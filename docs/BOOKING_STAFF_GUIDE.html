1. System Overview / ภาพรวมระบบ

The TestD booking system allows clients to schedule appointments at any of the four SWING branches. It supports both registered users and guest users (no account required).

Key Features

  • Multi-service appointments — Clients can select multiple services per visit
  • Density-aware scheduling — Time slots show occupancy levels (Low/Medium/High) to distribute load
  • Wait-time estimates — Considers counselor count, service weights, and walk-in pressure
  • Referral codes — Every booking gets a unique code (e.g. SWG-A3K9X2) for check-in
  • Real-time updates — Staff dashboard updates instantly when bookings are made or changed
  • Guest booking — No account needed; only an email is required

2. User Booking Flow / ขั้นตอนการจอง

This is what clients see when they visit /booking:

1 Choose Branch / เลือกสาขา

Client selects from 4 branches. Each shows a photo, Google Maps rating, address, and open hours.

2 Select Services / เลือกบริการ

Multi-select from available services (HIV test, PrEP, PEP, STI, etc.). Pricing eligibility shown automatically.

3 Pick Date & Time / เลือกวันและเวลา

Calendar + density-aware time blocks. Slots are color-coded: 🟢 Low, 🟡 Medium, 🔴 High occupancy.

4 Confirm & Get Code / ยืนยันและรับรหัส

Booking confirmed → unique referral code displayed. Client instructed to screenshot for check-in.

💡 Smart Suggestion / คำแนะนำอัจฉริยะ If a client picks a high-density time block, the system automatically suggests quieter alternatives to balance the load across the day.

3. Guest (Anonymous) Booking / การจองแบบไม่สมัครสมาชิก

Clients do not need an account to book. The only requirement is an email address.

How It Works:

  1. Client goes through the same booking flow (branch → services → date/time)
  2. Instead of login, they enter a contact email
  3. System creates the appointment securely via a server-side function
  4. Client receives a referral code on the success screen
✅ Account Linking / เชื่อมบัญชี After booking, guests are prompted: "อยากจัดการนัดง่ายขึ้นไหม? สมัครสมาชิกเพื่อดู เลื่อน หรือยกเลิกนัดได้ทันที"
If they register with the same email, the system offers to link the anonymous booking to their new account.

4. Referral Codes / รหัสอ้างอิง

📌 Format: SWG-XXXXXX
Auto-generated by database trigger. Unique per appointment.

For Clients / สำหรับผู้ใช้บริการ:

  • Displayed prominently on the booking success screen
  • Instruction shown: "กรุณาแคปหน้าจอนี้และแสดงให้เจ้าหน้าที่ลงทะเบียน"
  • Translation: "Please take a screenshot and show this to Medical Registration"

For Staff / สำหรับเจ้าหน้าที่:

  • Go to Admin → Booking tab
  • Use the referral code search box to find the booking instantly
  • This bypasses date filters — works even if you're viewing a different date
  • Verify client identity → update status to Arrived
⚠️ Important / สำคัญ Always verify the referral code matches the person. Ask the client to show their screenshot or email confirmation.

5. Staff Dashboard / แดชบอร์ดเจ้าหน้าที่

Access via: Admin → Booking tab (requires admin or staff role)

5.1 Bento Dashboard (Daily Overview)

The main view shows today's metrics at a glance:

MetricDescription
TotalAll appointments for the day
NewFirst-time clients
ReturningClients with prior non-cancelled visits
CancelledAppointments cancelled for the day

Each branch shows a load indicator so you can see which locations are busiest.

5.2 Branch Timeline

  • Click a branch card → opens a vertical timeline of all appointments
  • Each appointment shows as a pill with status badge and service icons
  • Click any pill → opens the Appointment Detail Drawer

5.3 Appointment Detail Drawer

From here, staff can:

  • View full appointment details (services, time, client info)
  • Update status (Confirmed → Arrived → In Progress → Completed)
  • Assign a staff member to the appointment
  • Add internal notes (visible to staff only)

5.4 Batch Actions

  • Select multiple appointments using checkboxes
  • Bulk status update — e.g., mark 5 as "Arrived" at once
  • CSV export — Download selected appointments for reporting

6. Walk-in Management / การจัดการ Walk-in

For clients who arrive without a booking:

1 Create Walk-in / สร้าง Walk-in

Staff clicks "New Walk-in" in the dashboard. Selects branch and adds optional notes. System creates appointment with source: 'walkin'.

2 Start Service / เริ่มบริการ

When counselor is ready, click "Start Service". Records arrival time.

3 Complete Service / เสร็จสิ้นบริการ

After consultation, click "Complete". Records completion time.

4 Impact on Estimates

Active walk-ins increase the walk-in pressure metric, causing wait-time estimates to go up for other clients.

💡 Quick Register / ลงทะเบียนด่วน The Admin panel also has a "Quick Register" tab where staff can register walk-in clients manually or via Thai ID OCR scan — no client device or internet needed.

10. Quick Reference Card / บัตรอ้างอิงด่วน

🏥 Daily Workflow for Registration Staff / ขั้นตอนประจำวัน
  1. Open Admin → Booking at the start of each day
  2. Check the Bento Dashboard for today's total appointments per branch
  3. When client arrives → ask for referral code or screenshot
  4. Search code in dashboard → verify details → mark as Arrived
  5. If walk-in → click "New Walk-in" → Start Service when ready
  6. After consultation → mark as Completed
  7. End of day → review any no-shows and update status
🔍 Finding a Booking Quickly
  • By referral code: Use the search box — bypasses date filter
  • By date: Use the calendar to navigate to the appointment date
  • By branch: Click the branch card to see its timeline
